# 开发文档

---

## 一、环境要求

### 最低要求
- **编译器**：g++ 支持 C++11 及以上
- **操作系统**：Windows 10+ / Linux (Ubuntu 18.04+)
- **终端**：支持 ANSI 转义序列的终端（Windows 10+ 自带，Linux 所有）
- **Make工具**：GNU Make 3.8+

### 推荐配置
- Windows 10+ 使用 **Windows Terminal**（支持ANSI颜色）
- Linux 使用任意现代终端（gnome-terminal / konsole / xterm）
- 字体：等宽字体（Consolas / Courier New / Monaco）

### 2.3 清理编译产物

```bash

```

### 2.4 常见编译问题

**问题1：`command not found: make`**
- **Windows**：安装 MinGW 或使用 Git Bash
- **Linux**：`sudo apt install build-essential`

**问题2：`undefined reference to Beep`**
- 原因：Linux 没有 Beep API
- 解决：Sound.cpp 中用 `#ifdef _WIN32` 区分平台

**问题3：中文乱码**
- 原因：终端编码不是 UTF-8
- 解决：`chcp 65001`（Windows）或设置终端编码为 UTF-8

## 三、代码规范

### 3.1 命名规范

```cpp
// 类名：大驼峰
class Monster { };
class AchievementManager { };

// 函数名：小驼峰
void updateAtb(float dt);
bool isAtbFull() const;

// 成员变量：小驼峰 + 下划线后缀
int hp_;
float atbGauge_;

// 常量：全大写 + 下划线
const int MAX_FLOORS = 20;
const int MONSTERS_PER_FLOOR = 8;

// 枚举：大驼峰
enum class TrapType { SPIKE, POISON, TELEPORT };
```

### 3.2 文件组织

```
头文件（.h）：
- 仅包含类定义、函数声明
- 用 #ifndef 防止重复包含
- 公共接口放 public，实现细节放 private

源文件（.cpp）：
- 包含具体实现
- #include 对应的 .h
- 不在头文件中暴露实现细节
```

### 3.3 注释规范

```cpp
// 函数注释：说明功能、参数、返回值
/**
 * 计算伤害值
 * @param atk 攻击力
 * @param def 防御力
 * @return 实际伤害（至少1点）
 */
int calcDamage(int atk, int def);

// 重要逻辑：加行内注释
if (atbGauge_ >= 100.0f) {
    // ⭐ ATB满槽，可以行动
    return true;
}

// TODO标记：标注待实现部分
void trigger(Player& player) {
    // TODO: A实现 - 根据 type_ 执行不同效果
}
```

## 四、Git 工作流

### 4.1 分支策略

```
main                    ← 主分支（稳定版本）
├── feature/A-map       ← 成员A：地图系统
├── feature/B-combat    ← 成员B：战斗系统
├── feature/C-render    ← 成员C：渲染系统
└── feature/D-game      ← 成员D：游戏系统
```

### 4.2 工作流程

```bash
# 1. 克隆仓库
git clone https://github.com/你的用户名/dungeon-rl.git
cd dungeon-rl

# 2. 创建自己的分支
git checkout -b feature/A-map  # A的分支
# git checkout -b feature/B-combat  # B的分支
# git checkout -b feature/C-render  # C的分支
# git checkout -b feature/D-game    # D的分支

# 3. 开发并提交
git add src/Map.cpp src/Map.h
git commit -m "[Map] 完成BSP地图生成算法"

# 4. 推送到远程
git push origin feature/A-map

# 5. 定期同步主分支（避免冲突）
git checkout main
git pull origin main
git checkout feature/A-map
git merge main
```

### 4.3 提交信息规范


## 五、调试技巧

### 5.1 打印调试

```cpp
#include <iostream>

// 调试输出
std::cout << "[DEBUG] ATB槽: " << atbGauge_ << std::endl;
```

### 5.2 使用 GDB

```bash
# 编译时加 -g 参数（Makefile已设置）
make

# 启动调试
gdb ./dungeon.exe

# GDB常用命令
(gdb) break main           # 在main设置断点
(gdb) run                  # 运行程序
(gdb) next                 # 单步执行
(gdb) print atbGauge_      # 打印变量
(gdb) continue             # 继续执行
```

### 5.3 常见问题排查

**战斗界面不显示速度条**
- 检查：`Entity::updateAtb()` 是否被调用
- 检查：`Render::drawBattleScreen()` 是否调用了 `drawATBBar()`

**怪物不移动**
- 检查：`Game::processMonsters()` 是否被调用
- 检查：怪物的 `takeTurn()` 实现是否正确

**颜色不显示**
- Windows：检查是否调用了 `enableANSI()`
- Linux：终端是否支持ANSI（几乎都支持）

## 六、性能优化建议

### 6.1 减少不必要的重绘

```cpp
// 不好：每帧都重绘
while (true) {
    render->drawAll(...);  // 每次都清屏+重画
}

// 好：只在状态变化时重绘
while (true) {
    if (stateChanged) {
        render->drawAll(...);
        stateChanged = false;
    }
}
```

### 6.2 避免频繁内存分配

```cpp
// 不好：每次都new
for (int i = 0; i < 100; i++) {
    Monster* m = new Monster(...);
    monsters.push_back(m);
}

// 好：预分配容量
monsters.reserve(100);
for (int i = 0; i < 100; i++) {
    monsters.push_back(new Monster(...));
}
```

### 6.3 使用引用避免拷贝

```cpp
// 不好：值传递，会拷贝整个对象
void processMonster(Monster monster) { ... }

// 好：引用传递，不拷贝
void processMonster(Monster& monster) { ... }
void processMonster(const Monster& monster) { ... }  // 只读
```
