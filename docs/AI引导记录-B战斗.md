### 日期: 第18周 6月6日
### 任务: Boss 回血机制 + ATB 测试完善

**我给了AI什么prompt：**
> 读取项目主干和枝叶B规格，审查现有代码，找出 B 模块缺失的功能。发现了 Boss 在战斗中不会回血的问题。

**AI输出了什么：**
- 在 Monster 基类中添加了 `virtual void onBattleTick()` 战斗回合回调
- Boss 重写 `onBattleTick()`，每 3 回合 `heal(5)`
- Combat 添加了 `static void onBattleTurn(Monster&)` 供 Game 调用
- test_combat.cpp 新增测试 9/10/11：ATB 速度条、Boss 回血、DeathKnight 无回血

**我发现了什么问题：**
- Boss 回血测试失败——Boss 初始满血 50/50，`heal(5)` 被 `maxHp_` 上限钳制导致 HP 不变

**我手动怎么改的：**
- 测试中先 `boss.takeDamage(20)` 把 Boss 打到 30 血，再测回血
- 增加了"回血不超过上限"的测试用例

**为什么这样改：**
- 满血无法验证回血效果，必须先受伤。上限测试确保 `Entity::heal()` 的 `if (hp_ > maxHp_)` 逻辑有效

### 日期: 第18周 6月6日
### 任务: 中文输入法兼容修复

**我给了AI什么prompt：**
> 中文输入法的时候会导致游戏无法操作，调一下让他能够在中文输入法也能操作，最好不要让他弹出中文框。

**AI输出了什么：**
- 在 `Input::Input()` 构造函数中加入 `ImmAssociateContext(GetConsoleWindow(), NULL)` 禁用控制台 IME
- `ReadConsoleInput` 改为 `ReadConsoleInputW`（宽字符版）
- 默认按键分支限制只接受 ASCII 可打印字符 (32-126)
- 新增 7/8/9/0 键映射支持商店买技能书

**我发现了什么问题：**
- 编译链接需要 `imm32.lib`，通过 `#pragma comment(lib, "imm32.lib")` 解决

**我手动怎么改的：**
- 无手动修改

**为什么这样改：**
- Windows 中文输入法会拦截控制台键盘事件，导致 WASD 等按键变成拼音输入
- `ImmAssociateContext` 是 Windows API 提供的标准方案，直接在游戏启动时关闭 IME

### 日期: 第18周 6月6日
### 任务: 编译错误排查 — Render.cpp Git 合并冲突

**我给了AI什么prompt：**
> 我更改了 Render.cpp 和 Render.h，现在编译不了，怎么回事？

**AI输出了什么：**
- 指出 Render.cpp 中存在 Git 合并冲突标记 `<<<<<<< Updated upstream` / `=======` / `>>>>>>> Stashed changes`
- 冲突原因：队友的双缓冲 ostringstream 版本和我的 std::cout 技能栏版本 Git 无法自动合并

**我发现了什么问题：**
- Git merge/pull 操作后没有检查冲突标记就直接保存了文件
- 编译器看到 `<<<<<<<` 不认识，直接报错

**我手动怎么改的：**
- 手动删除冲突标记，保留双缓冲架构，将技能栏改写为流式输出

**为什么这样改：**
- 双缓冲渲染效果更好（不闪烁），应保留
- 需要养成 `git pull` 后先搜索 `<<<<<<<` 的习惯
